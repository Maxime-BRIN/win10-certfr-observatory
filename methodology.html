Méthodologie de l'observatoire Windows 10 / CERT-FR

Cette page décrit comment le jeu de données est construit à partir des publications du CERT-FR, son périmètre et ses principales limites.

Périmètre fonctionnel

L'observatoire se concentre sur les vulnérabilités Windows 10 référencées dans des avis et bulletins d'actualité publiés par le CERT-FR après la fin de support officielle de Windows 10 (14 octobre 2025). Seules les pages listées explicitement dans le script de collecte sont analysées.

Sources CERT-FR utilisées

Les données sont extraites automatiquement à partir de différentes pages du site CERT-FR, en particulier :

  • certains avis de sécurité (par exemple CERTFR-2025-AVI-1092) pour lesquels des systèmes affectés incluent Windows 10 ;
  • plusieurs bulletins d'actualité (par exemple CERTFR-2025-ACT-045, CERTFR-2025-ACT-007, CERTFR-2026-ACT-007) contenant des tableaux de vulnérabilités avec colonnes Produit, CVE, score CVSS, type et exploitabilité ;
  • des contenus additionnels peuvent être ajoutés progressivement en étendant la liste de slugs dans le script.

Détection des vulnérabilités Windows 10

Le script recherche les entrées où la colonne Produit ou la section « Systèmes affectés » mentionne explicitement Windows 10 (par exemple « Microsoft Windows 10 Version 22H2 »). Les mentions génériques comme « Microsoft Windows » sans indication de version ne sont pas prises en compte afin de ne pas extrapoler.

Fenêtre temporelle post fin de support

Seules les CVE dont la date de publication se situe entre la fin de support de Windows 10 (14/10/2025) et la date de génération du jeu de données sont conservées. La date de publication est extraite à partir des colonnes de date présentes dans les tableaux des bulletins ou, à défaut, à partir des métadonnées de la page lorsque c'est possible.

Structure du jeu de données

Le fichier data/windows-10-certfr-data.json contient :

  • un bloc dataset avec le nom, la version, la date de génération et la fenêtre de couverture (start / end) ;
  • un tableau cves où chaque entrée représente une vulnérabilité identifiée par son identifiant CVE ;
  • pour chaque CVE : le score CVSS, le type d'impact, l'état d'exploitabilité, les versions Windows 10 concernées et les références CERT-FR associées.

Déduplication

Lorsqu'une même CVE apparaît dans plusieurs sources CERT-FR (avis, bulletins successifs), les informations sont fusionnées par identifiant CVE. Les listes de versions, de produits affectés et de références sont agrégées, et les informations de score ou d'exploitabilité les plus complètes sont conservées.

Limites et précautions d'usage

  • le périmètre n'inclut que les pages explicitement parcourues par le script ; des vulnérabilités Windows 10 peuvent exister dans d'autres publications non couvertes ;
  • les données restent dépendantes de la structure HTML actuelle des pages CERT-FR et peuvent devenir incomplètes si celle-ci évolue ;
  • ce jeu de données ne remplace ni un inventaire de parc, ni un scanner de vulnérabilités, ni une analyse de risque détaillée ;
  • les décisions de sécurité opérationnelle doivent s'appuyer sur plusieurs sources d'information, dont les avis officiels de l'éditeur et les politiques internes de gestion des risques.

Pour toute vérification ou analyse détaillée, il est recommandé de consulter directement les avis et bulletins sur le site du CERT-FR et de croiser ces informations avec les bulletins de l'éditeur (Microsoft) et vos inventaires internes.
